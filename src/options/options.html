Decant

Convert files to Markdown before they upload.

Where Decant runs

Decant does nothing on a site until you enable it here. Turning a host on asks Chrome for permission to run on that host.

    What happens to each file type

    Files are matched top-down by extension or MIME type; the first enabled rule wins and anything unmatched is sent untouched. A companion / endpoint rule POSTs the file to a local service; an in-browser rule can also escalate to one — but only when the browser finds no text (a scan).

      Config as JSON

      Power users: the whole config (hosts, rules, hotkey) as editable JSON. Rule fields beyond the form above — output name/type, error fallback, request encoding — are available here.

      Behavior

      The in-page prompt's “Set as default” checkbox changes this too. A choice that isn't possible for a file (no companion endpoint configured, no extractable figures) falls back to asking.

      Page capture

      The toolbar button (or Alt+Shift+C) converts the page you're reading to Markdown and attaches it in the chat you last used — the right-click menu picks a different chat. Reading the page needs no new site permissions: the click itself grants one-time access to that tab.

      Tokens saved

      Estimated total across every converted upload whose message was actually sent — drafts you abandoned don't count. An estimate for the same reasons the badge is one (~4 characters/token, a conservative per-page image cost).
